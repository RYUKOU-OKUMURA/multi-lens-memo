# 多視点メモ生成アプリ — 要件定義

作成日: 2026-03-30
ステータス: MVP 実装済み（要件は継続更新）

---

## 1. プロジェクト概要

| 項目 | 内容 |
|------|------|
| **プロダクト名（仮）** | Multi-Lens Memo（多視点メモ） |
| **目的** | 同一の入力テキストに対し、複数の「視点（レンズ）」ごとにAIがメモを並列生成し、多層的に読む思考の訓練と記録に使う |
| **一言で** | 天秤AI的UIで、同じ素材を3〜4つの視点から同時にメモ出力するWebアプリ |
| **利用者** | 自分専用（公開予定なし） |
| **背景** | 岡俊之『東大オタク入門』の「オタクは常に3つ・4つの目で見る」＋尾原さんの「疑似録を複数本」の話を、思考訓練ツールとしてプロダクト化する |

---

## 2. コア価値

- **やること**: 入力1つに対し、視点ごとに役割プロンプトを変えて、並列で短いメモ（解釈・着目点・問い）を生成する
- **やらないこと**: 文単位・段落単位の「どの箱に入れるか」分類（曖昧さが残りやすく、納得感の設計が難しい）
- **使い方の思想**: AI出力を正解として固定するのではなく、自分のメモや感想と並べて「差分」を見る鏡として使う

---

## 3. ユーザーフロー

### フローA（AI先行 — 自分メモ非表示）

1. 素材テキストを貼り付ける（文字起こし・マークダウン等）
2. 視点（レンズ）を確認・カスタマイズ
3. 自分メモカラムは非表示のまま、生成ボタンを押す
4. 左カラムにコンテキスト、右側に4視点のメモが横並びで同時にストリーミング表示される
5. 必要に応じて自分メモカラムをONにして書き足す
6. 全視点＋自分メモを1つのMDファイルとしてダウンロード → Obsidianに保存

### フローB（自分先行 — 自分メモ表示）

1. 素材テキストを貼り付ける
2. 自分メモカラムをONにし、まず自分のメモ・感想を書く
3. 生成ボタンを押すと、AIが自分の視点とは別の角度からメモを出す
4. 左からコンテキスト → 自分メモ → 4視点AI出力、の順で並び、比較しながら読める
5. 以降はフローAと同じ

---

## 4. 画面構成（UIレイアウト）

### 4.1 全体レイアウト

カンバン的なカラムレイアウト。1画面で素材・自分メモ・AI出力を横並びで一覧できる。

```
[自分メモ ON: 6カラム]
┌──────────┬───────┬───────┬───────┬───────┬───────┐
│          │自分メモ│ 当事者 │ﾃﾞｨﾚｸﾀｰ│ﾌﾟﾛﾃﾞｭｰｻｰ│ 歴史家 │
│コンテキスト│       │       │       │       │       │
│ (入力素材)│(入力) │(stream)│(stream)│(stream)│(stream)│
│          │       │       │       │       │       │
│ ※独立    │       │       │       │       │       │
│ スクロール │       │       │       │       │       │
└──────────┴───────┴───────┴───────┴───────┴───────┘

[自分メモ OFF: 5カラム]
┌──────────┬────────┬────────┬────────┬────────┐
│          │ 当事者  │ﾃﾞｨﾚｸﾀｰ│ﾌﾟﾛﾃﾞｭｰｻｰ│ 歴史家  │
│コンテキスト│        │        │        │        │
│ (入力素材)│(stream)│(stream)│(stream)│(stream)│
│          │        │        │        │        │
└──────────┴────────┴────────┴────────┴────────┘
```

### 4.2 各エリアの役割

| エリア | 説明 |
|--------|------|
| **コンテキスト（左端・固定）** | 素材テキストの表示。独立スクロール。長文（数万字）でも右側の視点カラムと干渉しない |
| **自分メモ（トグル表示）** | コンテキストの右隣に配置。ON/OFFトグルで表示切替。入力欄として自由記述 |
| **視点カラム × N（右側）** | 各レンズのAI出力をストリーミング表示。独立スクロール。各カラムにレンズ名がヘッダーとして表示 |

### 4.3 ヘッダー・操作エリア

```
┌───────────────────────────────────────────────┐
│ [レンズ設定]  [自分メモ ON/OFF]  [生成]  [MD DL] │
└───────────────────────────────────────────────┘
```

- **レンズ設定**: 視点の名前・説明を編集するモーダル or パネル
- **自分メモ ON/OFF**: 自分メモカラムの表示/非表示トグル
- **生成**: 全視点のメモ生成を同時開始
- **MD DL**: 全視点＋自分メモを1つのMDファイルとしてダウンロード

### 4.4 UIの原則

- **独立スクロール**: コンテキスト・各視点カラムはそれぞれ独立してスクロール可能（左の長文と右のメモで情報量の密度差が大きいため）
- **カラム幅の自動調整**: 自分メモのON/OFFで、視点カラムの幅が自動的に広がる/縮まる
- **エラー独立**: 1つの視点がエラーになっても他のカラムは継続表示

## 5. 機能要件

### 5.1 入力

- プレーンテキスト / Markdownのペースト（必須）
- 入力量: 1〜2時間分の文字起こし（2万〜5万字程度）に対応
- ファイルアップロードは将来検討。MVPはテキスト貼り付けのみ

### 5.2 視点（レンズ）

- **既定セット（4つ）**:
  - **当事者／観客**: 何が起きたか、感情・事実の表面
  - **ディレクター**: 構成・間・次に何が来そうか・演出上の意図の推測
  - **プロデューサー**: 制約・利害・布陣の理由の推測
  - **歴史家／文脈**: 系譜・過去の類似・参照元の推測
- **ユーザー定義**: 各レンズの名前＋説明（システムプロンプト断片）を編集・保存できる
- レンズ数: 3〜4を推奨（トークンコストのバランス＋カラム幅の視認性）

### 5.3 出力・表示

- **天秤AI的な横並びカラムUI**: 各視点が独立カラムに同時ストリーミング表示
- 各視点メモは3〜7行程度（短め推奨、「深掘り」アクションで長文化は将来検討）
- 自分メモカラム: トグルで表示/非表示を切替

### 5.4 エクスポート

- 全視点＋自分メモを **1つのMarkdownファイル** としてダウンロード
- 出力フォーマット例:

```markdown
# 多視点メモ: 〇〇
生成日: YYYY-MM-DD
素材: （素材タイトル or 冒頭抜粋）

---

## 自分のメモ
（自分で書いた内容。非表示・未記入の場合はセクション省略）

---

## 当事者／観客
- ...

## ディレクター
- ...

## プロデューサー
- ...

## 歴史家／文脈
- ...
```

### 5.5 オプション（将来）

- **クロスレビュー**: 視点Aのメモを視点Bの立場からコメントする（トークン増のため任意）
- **ファイルアップロード**: テキストファイル・MD直接読み込み

---

## 6. 非機能要件

| 区分 | 内容 |
|------|------|
| **利用形態** | 自分専用。認証不要 |
| **データ保存** | バックエンドDB不要。生成結果はMDダウンロードで管理 |
| **APIキー管理** | Cloudflare環境変数（サーバーサイド管理、ブラウザに露出しない） |
| **コスト可視性** | 長文×複数視点はトークンコストが嵩むため、推定コストをUI上に表示することを検討 |

---

## 7. 成功の定義（MVP）

- 1〜2時間分の文字起こしを入力し、4視点のメモが横並びで同時にストリーミング生成される
- 各視点のメモに意味の重なりと差分が読み取れる
- レンズ定義を差し替えても破綻せずに出力される
- 自分メモカラムのON/OFFが切り替えられる
- 1つのMDファイルとしてダウンロードし、Obsidianにそのまま保存できる
